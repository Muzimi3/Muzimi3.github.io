语音信号的读取与显示

​ 语音信号格式常用的是WAV格式，如果把WAV格式去掉信号的文件开头空白部分，只保留数据部分，则是PCM格式。一般的WAV格式文件，采用16bit均匀量化编码，即每2个字节表示一个采样点，采样点的取值范围-32768+32767，归一化为浮点表示-1+1。

语音信号预处理:

在数字信号处理（DSP）中特别是在时域到频域的变换以及特征提取过程中，经常会使用帧（frame）和帧移（hop size）的概念。这些概念在短时傅里叶变换（STFT）、滤波、语音处理等操作中都很常见。具体来说，理解为什么帧的个数frame_num = 总样本数N / 重叠数hop_size，首先需要明确几个基本概念：

  1. 帧（Frame）：在DSP中，信号通常被分割成许多小的窗口（即帧），每个帧包含一部分连续的样本。帧的长度通常称为frame_size（或window_size）。
  2. 帧移（Hop Size）：帧之间的移动步长，即相邻帧之间的距离。假设frame_size为M，hop_size为H，这意味着每个新的帧从前一个帧的位置向右移动H个样本。
  3. 重叠（Overlap）：相邻两个帧之间的重叠部分。重叠量等于frame_size - hop_size。

公式推导

假设我们有一个总长度为N的信号，我们要对其进行分帧处理，帧长度为frame_size，帧移为hop_size。

  1. 首帧位置：第一个帧从样本0开始，到样本(frame_size-1)结束。
  2. 第二帧位置：第二个帧从样本hop_size开始，到样本(hop_size + frame_size - 1)结束。
  3. 一般帧位置：第k个帧从样本k * hop_size开始，到样本(k * hop_size + frame_size - 1)结束。

为了覆盖整个信号，总共有N个样本，最后一个有效帧必须在信号末尾之前开始。因此，我们需要找到最大的整数k，使得：

k × hop_size + frame_size − 1 < N

这可以简化为：

k × hop_size + frame_size ≤ N

k × hop_size ≤ N − frame_size

由于k必须是整数，所以：

k ≤ ( N − frame_size ) / hop_size

因此，帧的总数（即frame_num）可以表示为：

frame_num = ⌊ ( N − frame_size ) / hop_size ⌋ + 1 = k+1

不过，在一些简化的表示和特定情况下，假设frame_size和hop_size的比值（以及重叠量）相对固定且处理较为标准化的信号段，frame_num可以近似表示为：

frame_num ≈ N / hop_size

这种简化通常是在理论说明中，假设重叠部分与帧移比较合理，信号段较长且边缘效应可以忽略的时候提出的。

解释

  • 概念上的一致性：每次帧移hop_size样本，因此为了覆盖总共N个样本，需要约N/hop_size的帧。
  • 边缘效应：实际应用中，边界处理（例如零填充）和边缘效应可能稍微调整实际的帧数，但近似公式仍然提供了一个有用的概念框架。

实际应用

在实际编程和信号处理任务中，具体实现会考虑边界条件、零填充和精确的帧计数，因此frame_num = N/hop_size是一个理论近似，实际应用中可能需要更为精细的计算和调整。
